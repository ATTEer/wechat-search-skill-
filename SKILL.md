# 微信公众号文章采集技能

## 配置说明

**第一步：定位技能目录。** 确定本 SKILL.md 文件所在目录的绝对路径，记为 `SKILL_DIR`。后续所有脚本调用和文件读取均以此目录为基准。

**第二步：加载配置。** 读取 `SKILL_DIR/config.yaml` 文件，加载所有配置值。配置项的含义和默认值以该文件为准，不要假设或硬编码任何配置值。在整个执行流程中使用从 `config.yaml` 中读取到的实际值。用户可能已自定义了部分配置。

## 执行流程

### 阶段一：解析用户意图

分析用户输入，确定以下信息：

1. **搜索模式**：
   - 如果用户提到了具体的公众号名称（如"采集公众号 人民日报"、"人民日报 的文章"）→ `mode = account`
   - 如果用户提供了主题/标题关键词（如"微信搜索 AI 大模型"、"搜索关于量子计算的文章"）→ `mode = title`
   - 如果无法判断，询问用户："请问您是要按文章标题搜索，还是按公众号名称搜索？"

2. **关键词**：从用户输入中提取搜索关键词。

向用户确认：
```
搜索模式: [标题搜索 / 公众号搜索]
关键词: [keyword]
```

等用户确认后再继续。

### 阶段二：搜索并采集 URL

运行搜索脚本（注意：必须先 cd 到技能目录 `SKILL_DIR`）：

```bash
cd SKILL_DIR/scripts && python search.py --mode [title|account] --keyword "[keyword]" --config SKILL_DIR/config.yaml
```

> 将 `SKILL_DIR` 替换为第一步中确定的技能目录绝对路径。

**重要：脚本内部已内置双模式浏览器连接，无需 AI 干预浏览器启动过程。**

**监控脚本输出（按顺序可能出现）：**

| 输出信号 | 含义 | AI 行为 |
|---------|------|---------|
| `[INFO] 已通过 CDP 连接到 Chrome` | CDP 连接成功 | 继续等待搜索完成 |
| `[WARN] CDP 连接失败` | CDP 不可用，正在自动回退 | 告知用户"浏览器正在自动启动（首次约需 30-60 秒）"，**继续等待，不要中止脚本** |
| `[INFO] 等待浏览器启动中...` | 浏览器启动心跳（每 5 秒一条） | 正常现象，**保持等待，不要重试或中止** |
| `[INFO] 浏览器窗口已启动` | 自动启动 Chromium 成功 | 继续等待搜索完成 |

**退出码处理（脚本结束后）：**

| 退出码 | 条件 | AI 行为 |
|-------|------|---------|
| `0` | 成功 | 读取 urls.json 报告结果 |
| `1` | 输出含 `playwright install chromium` | **停止，不要重试**，告知用户："浏览器自动启动失败，请在终端运行 `playwright install chromium` 后重试" |
| `1` | 其他原因 | **停止，不要重试**，将完整错误信息展示给用户，等待指示 |
| `2` | 无搜索结果 | 告知用户未找到结果，询问是否更换关键词 |

> ⚠️ **禁止在 search.py 退出后自动重试**。任何退出码非 0 的情况，必须先向用户报告再等待指示。
- 脚本暂停等待验证码：通知用户在浏览器中手动完成验证码，然后在终端按 Enter 继续。

**搜索完成后：**
读取 `{output_dir}/urls.json`，向用户报告：
```
搜索完成！共采集到 [N] 篇文章的链接。
是否继续下载文章内容和图片？
```

等用户确认后再进入阶段三。

### 阶段三：获取文章内容和图片

运行采集脚本（注意：必须先 cd 到技能目录 `SKILL_DIR`）：

```bash
cd SKILL_DIR/scripts && python fetch.py --config SKILL_DIR/config.yaml
```

> 将 `SKILL_DIR` 替换为第一步中确定的技能目录绝对路径。

**监控脚本输出：**
- 脚本会逐篇打印处理进度（如 "Processing 15/200..."）
- 完成后报告成功/失败数量。

**采集完成后：**
向用户报告：
```
文章内容采集完成！
成功: [N] 篇
失败: [M] 篇
图片已保存到: {output_dir}/{attachments_dir}/
```

### 阶段四：转换为 Obsidian Markdown

此阶段由 **Python 脚本 `convert.py` 自动完成**，无需 LLM 参与。

运行转换脚本（注意：必须先 cd 到技能目录 `SKILL_DIR`）：

```bash
cd SKILL_DIR/scripts && python convert.py --config SKILL_DIR/config.yaml
```

> 将 `SKILL_DIR` 替换为第一步中确定的技能目录绝对路径。

**脚本功能：**
- 读取 `{output_dir}/cleaned/` 下所有 JSON 文件
- 使用 `markdownify` 将 HTML 自动转换为 Markdown
- 生成 YAML frontmatter（title / author / date / source）
- 根据 `embed_images` 配置处理图片（`false` = 相对路径，`true` = 使用预编码的 Base64 Data URL）
- 按文件名规则（日期前缀 + 标题截断 + 去非法字符 + 重名追加后缀）写入 `.md` 文件

**监控脚本输出：**

| 输出信号 | 含义 |
|---------|------|
| `[OK] 2026-04-15-文章标题.md` | 单篇转换成功 |
| `[WARN] Skipping malformed file: 0.json` | 该 JSON 解析失败，已跳过，继续处理其余文章 |
| `[DONE] Converted N articles: N success, M failed` | 全部完成，附成功/失败统计 |

**退出码处理：**

| 退出码 | 含义 | AI 行为 |
|-------|------|---------|
| `0` | 成功（含部分失败但全局不中止） | 读取输出目录报告结果 |
| 非 0 | 脚本级错误（如导入失败） | 展示完整错误信息给用户，等待指示 |

**转换完成后：**
读取 `{output_dir}/` 下的 `.md` 文件列表，向用户报告：
```
Markdown 转换完成！共生成 [N] 篇文章。
输出目录: {output_dir}/
```

> 💡 **可选 LLM 增强**：若需要对生成的 Markdown 进行内容校对、广告过滤优化或格式精修，
> 可在 convert.py 完成后，由 LLM 读取指定的 `.md` 文件进行手动增强处理。
> 这是纯可选步骤，不影响主流程。

### 阶段五：报告结果

全部批次完成后，向用户报告：

```
========================================
采集完成！

搜索关键词: [keyword]
搜索模式: [标题搜索 / 公众号搜索]

采集结果:
  - URL 采集: [N] 篇
  - 内容下载: [M] 篇成功, [F] 篇失败
  - MD 转换: [K] 篇完成

输出目录: {output_dir}/
文件列表:
  - YYYY-MM-DD-文章标题A.md
  - YYYY-MM-DD-文章标题B.md
  - ...
========================================
```

## 错误处理

### CDP 连接失败
如果脚本输出包含 `[WARN] CDP 连接失败`：
```
⚠️ 未检测到 Chrome 调试端口，脚本正在自动启动浏览器窗口，请稍候...
（首次启动可能需要 30-60 秒）
```
等待脚本继续运行，**无需手动干预**。

如果脚本以退出码 1 退出且输出包含 `playwright install chromium`：
```
❌ 无法自动启动浏览器。

请在终端运行以下命令安装 Playwright 浏览器，然后重试：
  playwright install chromium
```

### 验证码检测
如果脚本输出包含 `[CAPTCHA]`：
```
⚠️ 搜狗检测到频繁访问，需要验证码验证。

请在浏览器中完成验证码，然后在终端按 Enter 继续。
```

### 脚本执行出错
如果脚本意外失败：
```
❌ 脚本执行出错: [错误信息]

可能的解决方案:
1. 检查网络连接
2. 确认 Chrome 浏览器仍在运行
3. 检查 config.yaml 配置是否正确
4. 重试操作
```

### 部分失败
如果部分文章采集失败但其他成功，继续处理成功的文章，在最终报告中列出失败的条目。

## 禁止事项

以下是执行此技能时严格禁止的行为。违反任何一条都可能导致采集失败、账号封禁或数据丢失。遇到问题时，应向用户报告并等待指示，而非自行尝试绕过。

### 禁止手写代码替代脚本
- 禁止在执行过程中编写新的 Python 脚本、Node.js 脚本或任何代码来替代 `scripts/` 目录下的现有脚本
- 禁止用代码片段临时实现搜索、采集、图片下载等功能
- 如果现有脚本执行失败或无法满足需求，必须向用户报告问题并等待指示，而非自行编写替代代码
- 阶段四的 Markdown 转换是唯一由 AI 直接执行的任务，其他所有阶段必须通过现有脚本完成

### 禁止使用非 CDP 模式
- 禁止使用 `requests`、`httpx`、`urllib`、`curl` 等 HTTP 客户端直接请求搜狗或微信页面
- 禁止使用 `headless=True` 模式启动浏览器（需保持窗口可见，以便用户手动处理验证码）
- 禁止使用 Selenium、Puppeteer 或其他非 Playwright 方案

### 禁止自动处理验证码
- 禁止使用 OCR、第三方验证码识别服务或任何自动化手段处理验证码
- 遇到验证码必须暂停并通知用户手动处理

### 禁止自动重试脚本
- 禁止在 search.py 或 fetch.py 退出后（无论任何退出码）自动重新运行该脚本
- 任何非 0 退出码必须先向用户报告完整错误信息，等待用户指示后才能重试
- 脚本运行期间若输出 `[WARN]` 日志，说明内部正在处理回退，不得中止脚本

### 禁止修改现有脚本
- 禁止在执行过程中修改 `scripts/search.py`、`scripts/fetch.py`、`scripts/utils.py` 的代码
- 如果脚本存在 bug 或需要更新，应向用户报告，由用户决定是否修改

### 禁止跳过用户确认
- 阶段二搜索完成后，必须等待用户确认才能进入阶段三
- 禁止在用户未确认的情况下自动开始下载文章内容

### 禁止直接访问微信文章 URL
- 禁止用 HTTP 请求直接访问 `mp.weixin.qq.com` 的文章链接
- 所有文章访问必须通过 CDP 控制的浏览器完成，以确保正确的 Cookie、Referer 和反防盗链处理
